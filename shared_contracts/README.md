# Shared Contracts

Canonical trading contracts shared between `crypto-ai-bot` and `signals-api`.

## Overview

This package defines the **"System Law"** objects that enforce a single, consistent pipeline across the entire trading system:

```
Strategy → TradeIntent → ExecutionDecision → Trade
```

## Installation

```bash
# In crypto-ai-bot environment
conda activate crypto-bot
pip install -e ../shared_contracts

# In signals-api environment
conda activate signals-api
pip install -e ../shared_contracts
```

## Canonical Models

### Strategy
Immutable configuration for signal generation.
- Defines parameters, timeframes, risk profile
- No execution logic, just configuration

### TradeIntent
What the system intends to do (pre-risk evaluation).
- Generated by strategy + market data
- **Must include reasons** for explainability
- Links to strategy_id

### ExecutionDecision
Result of risk evaluation on a TradeIntent.
- Approved or Rejected
- If rejected, must explain why
- Captures risk_snapshot at decision time

### Trade
Final execution record.
- **Can only exist if ExecutionDecision was approved**
- Links back to full explainability chain
- Tracks fills, slippage, fees, P&L

    def execute(self, decision: ExecutionDecision):
        # Execute only if decision.is_approved

## Non-Negotiables

1. **No execution without approved ExecutionDecision**
2. **All models are immutable** (frozen=True)
3. **Explainability required**: reasons, inputs, risk context
4. **Same pipeline for backtest and paper/live** (parity)
